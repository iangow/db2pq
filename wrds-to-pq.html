WRDS to Parquet

Use this path when the source data live in WRDS PostgreSQL and the destination is a local Parquet file repository.

Main helpers

  • wrds_update_pq(table_name, schema, ...) only rewrites a Parquet file when the WRDS source is newer.
  • wrds_sql_to_pq(sql, table_name, schema, ...) exports a custom WRDS SQL query into the standard Parquet layout.
  • wrds_update_schema(schema, ...) updates a full WRDS schema into Parquet.
  • wrds_pg_to_pq(table_name, schema, ...) exports one WRDS table to Parquet, but without the update logic of wrds_update_pq().

Typical first run

from db2pq import wrds_update_pq

wrds_update_pq("dsi", "crsp")

If WRDS_ID is not configured yet, db2pq can prompt for it and suggest adding it to a local .env file. If the WRDS PostgreSQL password is not yet in .pgpass, db2pq can prompt for it and save it for later use.

More examples

A CRSP-style export where identifier columns are normalized to integer types:

from db2pq import wrds_update_pq

wrds_update_pq(
    "ccmxpf_lnkhist",
    "crsp",
    col_types={"lpermno": "int32", "lpermco": "int32"},
)

Using SAS metadata with use_sas=True

Use this path when the WRDS PostgreSQL table comment is missing or not the freshness signal you want. In this mode, db2pq checks SAS-side metadata instead.

A RavenPack-style example where the WRDS PostgreSQL schema and the SAS metadata schema do not line up:

from db2pq import wrds_update_pq

wrds_update_pq(
    "rpa_entity_mappings",
    "ravenpack_common",
    use_sas=True,
    sas_schema="rpa",
)

You can also combine use_sas=True with explicit timestamp coercions when exporting annual RavenPack tables:

wrds_update_pq(
    "rpa_djpr_equities_2024",
    "ravenpack_dj",
    use_sas=True,
    sas_schema="rpa",
    batched=False,
    col_types={
        "timestamp_utc": "timestamp",
        "rpa_time_utc": "string",
        "event_start_date_utc": "timestamp",
        "event_end_date_utc": "timestamp",
    },
)

Common options

  • engine="duckdb" uses DuckDB to read from PostgreSQL.
  • engine="adbc" streams Arrow batches directly from PostgreSQL.
  • where= applies a SQL filter before export.
  • keep= and drop= filter source columns with regex patterns.
  • obs= limits rows for testing.
  • archive=True moves replaced Parquet files into an archive directory.
  • use_sas=True can use SAS metadata for update detection when PostgreSQL comments are missing or incomplete.

Output layout

Files are written as:

<DATA_DIR>/<schema>/<table>.parquet

When To Use The Web Query

The WRDS web query can be useful when you are trying to identify the exact table behind a dataset or want WRDS to draft a starting SQL query for you. That is often a practical way to move from the WRDS website to a wrds_sql_to_pq(...) workflow.
